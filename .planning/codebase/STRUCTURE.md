## Where to Add New Code

**New Homepage Section:**
- Add HTML section to `index.html` with `id` attribute for hash navigation
- Add styles to `css/obsidian.css` following the numbered section pattern (e.g., `/* 8. NEW SECTION */`)
- Add interactivity to `js/site.js` following the IIFE pattern with a named `initXxx()` function
- If data-driven, add data to `content/data.json`

**New Career Content Article:**
- Create HTML file at `career-content/articles/{slug}.html`
- Follow the template pattern from existing articles: set `PAGE_TITLE`, `PAGE_DESCRIPTION`, `REL_PATH`, `CANONICAL_PATH` variables in inline `<script>`
- Link shared CSS via `../css/style.min.css` (or `../css/style.css`)
- Link shared JS via `../js/loader.min.js` and `../js/main.min.js`
- Add entry to `sitemap.xml`

**New Career Content Case Study:**
- Create HTML file at `career-content/case-studies/{slug}.html`
- Same template pattern as articles
- Add link card to `index.html` in the portfolio section
- Add entry to `sitemap.xml`

**New Interactive Tool (Resource):**
- Create self-contained HTML at `career-content/resources/{slug}.html`
- Embed ALL CSS and JS inline (resources pages are fully standalone)
- Add link to homepage POCs section in `index.html`
- Add entry to `sitemap.xml`

**New Shared CSS for Career Content:**
- Add to `career-content/css/` directory
- Import via `<link>` in each page that needs it

**New Image:**
- Homepage images: `images/`
- Section-specific images: `career-content/{section}/images/`
